## Goal

Build a single Rust binary (`ansible-password-agent`) that acts as a secure credential provider for Ansible's `--vault-password-file` and `--become-password-file` scripts. Secrets never touch disk in cleartext.

## Architecture

```
src/
ÔöťÔöÇÔöÇ main.rs            # CLI entrypoint, arg parsing, orchestration
ÔöťÔöÇÔöÇ backend/
Ôöé   ÔöťÔöÇÔöÇ mod.rs         # trait PasswordBackend + dispatch
Ôöé   ÔöťÔöÇÔöÇ linux.rs       # cfg(target_os = "linux") ÔÇö linux-keyutils
Ôöé   ÔööÔöÇÔöÇ macos.rs       # cfg(target_os = "macos") ÔÇö security-framework
ÔööÔöÇÔöÇ tty.rs             # rpassword prompt reading from /dev/tty
```

**Public interface** ÔÇö a single trait:
```rust
trait PasswordBackend {
    /// Retrieve cached password or None if missing/expired.
    fn get(key: &str) -> Result<Option<String>>;
    /// Store a password. On Linux this sets a 600s timeout.
    fn set(key: &str, secret: &str) -> Result<()>;
}
```

## CLI Interface

```
ansible-password-agent --type <vault|become>
```

- `--type vault` / `--type become` ÔÇö selects the cache key identifier (e.g. `ansible-vault-password` or `ansible-become-password`).
- If neither is given, default to `vault` for backward compatibility.

## Execution Flow (identical on both platforms)

1. Parse `--type` flag, derive the storage key string.
2. Call `backend::get(key)`.
3. If `Ok(Some(secret))` Ôćĺ print to stdout, exit 0.
4. If `Ok(None)` or error indicating missing item Ôćĺ prompt user via `/dev/tty` using `rpassword`.
5. If user cancels (Ctrl+C / empty input) Ôćĺ exit 1 (no stdout output).
6. Call `backend::set(key, &secret)`.
7. If set fails Ôćĺ exit 1 with stderr message.
8. Print secret to stdout, exit 0.

## Platform Backends

### Linux (`backend/linux.rs`)

- **Crate**: `linux-keyutils` (wraps libkeyutils)
- **Keyring**: user session keyring (`KEY_SPEC_USER_KEYRING`)
- **Key type**: `user` (simple key type for arbitrary byte payloads)
- **Key description**: `"ansible-password-agent:<type>"` (e.g. `ansible-password-agent:vault`)
- **Timeout**: 600 seconds on every `set()` and refreshed on every successful `get()`
- On `get()`: search by description. If found Ôćĺ `keyctl_update_timeout(req_key, 600)` Ôćĺ read payload Ôćĺ return. If not found Ôćĺ return `Ok(None)`.
- On `set()`: `add_key("user", desc, payload, KEY_SPEC_USER_KEYRING)` Ôćĺ `keyctl_set_timeout(req_key, 600)`.
- Payload stored as raw UTF-8 bytes (passwords are strings).

### macOS (`backend/macos.rs`)

- **Crate**: `security-framework`
- **Saving**: Build a `SecAccessControl` with `.kSecUseAuthenticationWithBiometrics` (Touch ID / Apple Watch / device password). Create a `SecItemAdd` call with:
  - `kSecClass`: `kSecClassGenericPassword`
  - `kSecAttrService`: `"ansible-password-agent"`
  - `kSecAttrAccount`: `<type>` (vault or become)
  - `kSecValueData`: password bytes
  - `kSecAttrAccessControl`: the biometric policy
- **Retrieving**: `SecItemCopyMatching` with service + account query. macOS presents Touch ID / password dialog automatically. On `errSecAuthFailed` or `errSecInteractionNotAllowed` Ôćĺ fall through to terminal prompt.
- **Error mapping**:
  - `errSecItemNotFound` Ôćĺ `Ok(None)` (first run, needs terminal prompt)
  - `errSecAuthFailed` / user canceled biometric Ôćĺ exit 1
  - `errSecInteractionNotAllowed` (e.g. no GUI session) Ôćĺ fall through to terminal prompt + still try to save with biometric policy (may fail, which is acceptable ÔÇö fall back to prompting every time)

## TTY Prompt (`tty.rs`)

- Use `rpassword::prompt_password_from_tty` or `rpassword::read_password_from_tty`.
- `rpassword` reads from `/dev/tty` by default, which is independent of Ansible's stdin redirection.
- Prompt message: `"Enter Ansible vault password: "` or `"Enter Ansible become password: "`.
- If the function returns an error or empty string Ôćĺ exit 1.

## Error Handling

- `anyhow::Result` throughout.
- Any backend error during `get()` that is *not* "item not found" Ôćĺ print to stderr, exit 1.
- Any backend error during `set()` Ôćĺ print to stderr, exit 1 (Ansible must not proceed with an uncached password that will be asked again needlessly ÔÇö though this is debatable; we'll exit 1 for safety).
- User cancellation Ôćĺ clean exit 1, no stdout.
