## Overview

Rewrite gitjuggling from a "run git commands in parallel" tool to a **repository sync tool** that reconciles local clones with GitHub/Forgejo upstreams. The old `exec` functionality is kept as a subcommand.

## Architecture

### Subcommands
- `gitjuggling sync [workspace]` — Main sync flow (default workspace if omitted)
- `gitjuggling exec <git args...>` — Old behavior: run a git command in all local repos

### Config File (`~/.config/gitjuggling/config.toml`, overridable with `--config`)

```toml
default_workspace = "personal"

[workspace.personal]
root = "/home/vincent/dev/perso"
github_owners = ["vrischmann"]
forgejo_url = "https://git.rischmann.fr"
forgejo_user = "vincent"
forgejo_token_cmd = "op read 'op://vault/item/field'"
local_scan_root = "/home/vincent/dev/perso"

[workspace.personal.rules]
base = "/home/vincent/dev/perso/repos"
forks = "/home/vincent/dev/perso/forks"
archived = "/home/vincent/dev/perso/archived"

[workspace.work]
root = "/home/vincent/dev/Batch"
github_owners = ["BatchLabs", "allyourcodebase"]
# No forgejo config = no Forgejo sync for this workspace

[workspace.work.rules]
base = "/home/vincent/dev/Batch/repos"
forks = "/home/vincent/dev/Batch/forks"
archived = "/home/vincent/dev/Batch/archived"
```

### Sync Flow

1. **Load config** for the selected workspace
2. **Fetch remote repos** from GitHub (via `gh auth token`) and Forgejo (via `op read`)
3. **Deduplicate**: skip Forgejo repos that are mirrors of GitHub repos
4. **Classify** each repo by rules: fork → archived → base
5. **Discover local clones** by recursively scanning `local_scan_root`, reading git remote URLs
6. **Match** local repos to upstream repos by remote URL
7. **Plan actions**:
   - **Already correct**: repo exists at the right path → `git stash -u && git pull --rebase`
   - **Needs move**: repo exists but at wrong path → propose move (interactive or dry-run)
   - **Needs clone**: repo doesn't exist locally → clone it
8. **Execute** actions (with progress bar, parallel where possible)

### Mirror Detection
- Forgejo API exposes a `mirror` flag on repo objects
- Skip any Forgejo repo where `mirror == true` (to avoid duplicating a GitHub clone)

### Rule Priority
1. Fork → `rules.forks` directory
2. Archived → `rules.archived` directory  
3. Default → `rules.base` directory

### Existing Repo Update Strategy
- `git stash -u` (include untracked)
- `git pull --rebase`
- On any failure: skip with warning, do not abort the whole sync

### Dependencies to Add
- `serde` + `toml` — Config parsing
- `reqwest` — GitHub/Forgejo API calls (or use `gh` CLI for GitHub)
- `dialoguer` or similar — Interactive y/n prompts for moves
- Remove: `indicatif` can stay, `colored` stays, `rayon` stays

### Error Handling
- Use `anyhow` for errors
- Individual repo failures should not abort the sync
- Summary at the end (like current tool)
